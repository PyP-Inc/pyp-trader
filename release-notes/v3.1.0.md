# PyP Trader v3.1.0

Stable public EA release focused on safer live execution, clearer routing, and better MetaTrader setup flow.

## Highlights

- Added explicit deployment-aware routing support for marketplace and live strategy execution
- Improved signal handling so newer EA builds do not blindly replay the latest signal after every restart
- Added chart-symbol filtering support to help prevent cross-chart execution mistakes
- Improved MT5 broker compatibility by using a broker-supported order filling mode instead of forcing a single mode
- Added better transport/error logging so MetaTrader-side request failures are easier to diagnose
- Aligned the EA flow with PyP dashboard-generated `.set` presets for easier user setup

## Execution Improvements

- EA now works more cleanly with PyP deployment-specific routing
- Supports deployment IDs provided from the PyP dashboard
- Preserves signal-processing state more safely across restarts
- Uses server execution payload values more consistently
- Enforces max concurrent trade limits in newer EA logic

## MT5 Improvements

- Removed reliance on a single hardcoded filling mode
- Improved compatibility with brokers that reject unsupported filling modes
- Improved order-close path logging and return-value checks

## Setup Experience

- PyP dashboard now supports `.set` preset downloads for subscription-based setups
- Non-technical users can use:
  - latest GitHub release binaries
  - dashboard-generated `.set` presets
- Technical users can still compile directly from source in MetaEditor

## Recommended Installation

1. Download the latest EA binary from the GitHub release
2. Configure your subscription or deployment in the PyP dashboard
3. Download the matching `.set` preset from PyP
4. Load the preset into MT4 or MT5
5. Allow WebRequests to:
   - `https://api.pyp.stanlink.online`

## Notes

- Older EA builds may still show less accurate transport or routing behavior
- If you are upgrading from older v3 builds, replace the EA binary rather than relying on an old compiled copy
- MT5 users should especially update to get the improved broker filling-mode behavior
